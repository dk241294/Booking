# Booking API Automation using Karate Framework

This repository contains automated tests for a Booking API using the Karate framework. Karate is a powerful tool for API testing. Follow the instructions below to get started.

## Prerequisites
Before running the tests, ensure that you have the following prerequisites installed:

- Java Development Kit
- Maven

## Running Tests
1. Clone this repository to your local machine. (https://github.com/dk241294/Booking/tree/main)

2. Open the project using your preferred text editor or Java IDE.

3. To run the tests, use one of the following methods:

   - Using Karate Runner Classes:
   - Navigate to the 'src/test/java/com/api/automation/runner' folder.
   - Run the Karate runner classes to execute the tests.

   - Using JUnit 5 Integration:
   - Refer to [Karate's JUnit 5 documentation](https://github.com/karatelabs/karate#junit-5) for details on running tests with JUnit 5.

   - Using Karate Plugin (IDE-specific):
   - Install the Karate plugin from your IDE's marketplace.
   - Open any feature file and click the run button.

## Test Reports
After running the tests, you can access the following test reports:

- Karate Default Report:
- Open 'target/karate-reports/karate-summary.html' in your web browser to view the Karate default report.

- Cucumber-style Report:
- If you prefer Cucumber-style reports, open 'target/cucumber-html-reports/overview-features.html' in your web browser to see the test results in a Cucumber-style format.

For more detailed information and advanced usage, refer to the [Karate documentation](https://github.com/karatelabs/karate#index).
